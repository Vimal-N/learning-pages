Why Opencode with Copilot?

If your organisation already pays for GitHub Copilot, you can run a terminal agent at no extra cost.

GitHub Copilot Enterprise and Copilot Individual subscriptions give you access to the same underlying models that power Copilot Chat. Opencode can use these models as its backend — meaning you get terminal agent capability using your existing licence, no Anthropic API key or separate billing required.

Claude Code
  • Requires an Anthropic API key or Claude Max subscription
  • Pay per token (or flat rate with Max)
  • Uses Claude Sonnet / Opus models — excellent for code
  • Best option if you have Anthropic billing
Opencode with Copilot backend
  • Uses your existing GitHub Copilot subscription
  • No separate API billing for Copilot Enterprise users
  • Open source — configurable, extensible
  • Best option if your org already has Copilot Enterprise

Check your licence first: Opencode using the Copilot backend works with GitHub Copilot Enterprise and GitHub Copilot Individual. If your organisation manages Copilot through a corporate licence with usage controls, check whether API usage via Opencode is permitted under your organisation's policy before using it.

Installation & Setup

Install Opencode and connect it to your GitHub Copilot account.

1
Install Opencode
Run: npm install -g opencode-ai. Requires Node.js 18+. Verify with opencode --version.
2
Authenticate with GitHub Copilot
Run opencode auth and select GitHub Copilot as your provider. Opencode will open a browser window for OAuth authentication. Log in with the GitHub account that has your Copilot subscription.
3
Select a model
Run opencode models to list the models available under your Copilot subscription. For QA automation tasks, gpt-4o or claude-3.5-sonnet (if available to your licence) are strong choices. Set your default: opencode config set model gpt-4o.
4
Navigate to your project and start
cd my-playwright-project then opencode. The interface is similar to Claude Code — an interactive prompt where you type tasks.
# Install
npm install -g opencode-ai

# Authenticate with GitHub Copilot
opencode auth

# Check available models
opencode models

# Set default model
opencode config set model gpt-4o

# Start a session
cd my-playwright-project
opencode

Model Selection — Picking the Right One

Opencode supports multiple models via the Copilot backend. Here's what to choose for QA work.

The models available to you depend on your GitHub Copilot subscription tier. The key choice is between GPT-4 class models and Claude models — both are available through Copilot Enterprise.

gpt-4o
OpenAI — strong all-rounder
Good at code generation and debugging. Widely available on Copilot subscriptions. Faster response times than larger models. Good default for most QA automation tasks.
claude-3.5-sonnet
Anthropic — via Copilot Enterprise
Available on Copilot Enterprise. Strong at long-context reasoning and multi-file tasks. If your licence includes it, this is often the best choice for complex test refactors or large suite generation.
o1 / o3-mini
OpenAI — reasoning models
Slower but stronger at complex multi-step reasoning. Best for architecture decisions or designing a test strategy, not routine test generation where speed matters more.
Routine test generation or debugging
gpt-4o — fast and accurate enough
Large multi-file refactor or complex test suite
claude-3.5-sonnet if available — better long-context reasoning
Designing a test strategy or reviewing architecture
o1 or o3-mini — worth the slower response time

Context Files with Opencode

Which context file Opencode reads — and how to configure it.

Opencode reads context files based on the backend you're using. When using the Copilot backend, it reads .github/copilot-instructions.md. When using the Anthropic backend, it reads CLAUDE.md.

→
Copilot backend — Opencode reads .github/copilot-instructions.md. This is the same file Copilot Agent Mode reads in VS Code, so if you already have one, Opencode picks it up automatically.
→
Anthropic backend — Opencode reads CLAUDE.md from your project root, same as Claude Code.
→
Both in one project — If your project might be used with both Claude Code and Opencode/Copilot, maintain both files. The content can be identical — just copied with different filenames.

Tip: If you already have a good .github/copilot-instructions.md from using Copilot Agent Mode, Opencode will pick it up immediately — no extra setup needed. The same project rules that help Copilot Agent Mode in VS Code help Opencode in the terminal.

Opencode vs Claude Code — Practical Differences

Both follow the same agentic loop. The practical differences come down to model access and UI.

Claude Code
  • Official Anthropic tool — tightly integrated with Claude models
  • Requires Anthropic API key or Claude Max
  • Permission model: prompts before running commands
  • Excellent at long context — handles large codebases well
  • More established, more community resources
Opencode
  • Open source — community-maintained and customisable
  • Multi-backend: Copilot, Anthropic, OpenAI, Gemini
  • Uses existing Copilot licence — potentially no extra billing
  • Slightly different UI — terminal TUI with session management
  • Growing community; docs and examples are expanding

Day-to-day: they're interchangeable for most tasks

For routine QA tasks — generating tests, debugging failures, refactoring — the choice of Claude Code vs Opencode is mostly a billing and preference decision. The agent loop, the GCSV prompt structure, and the git diff review workflow are identical. What you learned in Sessions 10–13 applies to both tools without modification.
