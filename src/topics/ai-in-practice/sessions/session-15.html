Step 1 — Getting your GitHub token

Recommended — GitHub CLI
Uses your existing GitHub login. No new token to create or manage.
# Verify GitHub CLI is logged in
gh auth status

# Export the token into your shell
export GITHUB_TOKEN=$(gh auth token)

# Confirm it's set (shows first 10 chars)
echo $GITHUB_TOKEN | head -c 10
The token starts with ghu_ — that's a GitHub user token, which is what you need.
Alternative — Personal Access Token
Create at github.com/settings/tokens — use a Fine-grained PAT.
1. Go to github.com/settings/tokens
2. Create a new Fine-grained token
3. Under Permissions → Repository → Copilot: Read & Write
4. Copy the token (starts with github_pat_)
export GITHUB_TOKEN=github_pat_xxxx
Add the export line to your ~/.zshrc or ~/.bashrc so you don't have to run it every time you open a terminal.

Step 2 — The Opencode config file, annotated

~/.config/opencode/config.json
{{ '{' }}                                                             ← A: root config object
  "provider": {{ '{' }}                                               ← A: provider block
    "github": {{ '{' }}                                             ← B: provider name
      "apiKey": "${GITHUB_TOKEN}"                           ← C: reads from env var
    {{ '}' }}
  {{ '}' }},
  "model": "github/claude-sonnet-4"                              ← D: provider/model-name
{{ '}' }}
B — "github": The provider name. Tells Opencode to use GitHub's Copilot API endpoint. Use "anthropic" for direct Claude, "openai" for direct GPT-4o.
C — apiKey: The ${GITHUB_TOKEN} syntax reads from your environment — your token never lives in this file as plain text.
D — model: Format is always provider/model-name. The "github/" prefix routes through Copilot's API rather than Anthropic's directly.
Tip: Run opencode models list to see every model name available through your configured provider.

Step 3 — Launch Opencode and start a session

Starting a session
# 1. Make sure your token is set
export GITHUB_TOKEN=$(gh auth token)

# 2. Go to your project
cd your-qa-project

# 3. Launch Opencode
opencode
Useful commands inside Opencode
/model Open the model picker — switch to a different Copilot model mid-session
/new Start a fresh session — resets context, keeps your config
/compact Summarise and compress the conversation to free up context window space
Ctrl+C Stop the current agent action (mid-stream, before it finishes)
/help List all available slash commands in the current version

Choosing the right Copilot model for your task

claude-sonnet-4
Default recommendation
Multi-file tasks, writing new test suites, refactoring existing tests, understanding complex code flows. Best reasoning quality — use for anything where accuracy matters more than speed.
gpt-4o
Fast iteration
Quick follow-up questions, small code changes, generating boilerplate, asking "how does this API work?" Fast responses make it good for when you're iterating quickly.
o1 / o3-mini
Hard problems only
Complex flaky test root-cause analysis, architectural decisions, debugging async race conditions. Significantly slower — justify the wait with a genuinely hard problem.
gemini-1.5-pro
Large codebases
When your test suite or codebase is too large for other models to hold in context — 1M token window means it can read everything at once. Use when context overflow is the problem.

Troubleshooting common issues

Error
Unauthorized / 401
Cause: GITHUB_TOKEN is missing, expired, or doesn't have Copilot access.
export GITHUB_TOKEN=$(gh auth token)  # re-run this, then retry
Error
Model not found
Cause: Typo in model ID, or that model isn't available on your Copilot plan.
opencode models list  # shows exact model IDs available
Error
No Copilot access / Forbidden
Cause: Your GitHub account doesn't have an active Copilot subscription, or the subscription doesn't include API access.
Check: github.com/settings/copilot — verify your plan is active and includes model API access.
Error
opencode: command not found
Cause: Opencode wasn't installed, or npm global bin isn't in your PATH.
npm install -g opencode-ai  # reinstall; then restart your terminal

Key takeaways

1
Your Copilot subscription is already a model API
Opencode gives you terminal agent access to Claude, GPT-4o, o1, and Gemini — all billed to the Copilot subscription you already have. No separate Anthropic account needed.
2
Setup is three steps: authenticate, configure, launch
GitHub CLI token → config.json with provider + model → opencode command. Everything from Sessions 10 and 11 (GCSV prompts, git checkpoints, staged tasks) works identically.
3
Multi-model flexibility is the unique advantage
Use claude-sonnet-4 for deep reasoning tasks, gpt-4o for fast iteration, o1/o3-mini for hard debugging, gemini-1.5-pro for very large codebases. Switch with /model mid-session.
4
It's the same model as Copilot Agent Mode — different interface
Copilot Agent Mode and Opencode+Copilot both use Claude Sonnet. The difference is IDE vs terminal workflow. Use whichever fits how you work — the results are equivalent.
