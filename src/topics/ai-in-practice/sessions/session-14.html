Session 14  ·  Phase 4: Terminal Agents & Agentic Tools

Comparing Your
AI Toolkit

You have five tools. Learn when to reach for each one — and how to combine them for maximum impact.

Intermediate
Level
Session 11
Prerequisite

Your full AI toolkit

Copilot Completions
Lives in your editor. Auto-suggests as you type. Inline, fast, no conversation required.
Copilot Chat
Lives in your IDE sidebar. Conversational. Can reference open files with @. Answers questions and generates code on demand.
Copilot Agent Mode
Chat upgraded. Can read files, edit files, run commands, self-correct. Goal-oriented. Lives in VS Code.
Claude Code
Terminal agent. Full codebase access. Shell command execution. Deepest reasoning. Paid API or subscription.
Opencode
Terminal agent with TUI. Open source. Model-agnostic. Useful when Claude or Copilot aren't available.

Side-by-side comparison

Tool Home Scope Best for Activation
Completions IDE Current file Inline completion while typing Automatic
Copilot Chat IDE Open files + context Questions, explanations, quick code gen Type in sidebar
Agent Mode IDE Project + shell Multi-file tasks, scaffolding, test suites Ctrl+I, give goal
Claude Code Terminal Full system Complex, exploratory, long-running tasks claude in terminal
Opencode Terminal Full system Same as Claude Code; model flexibility opencode in terminal

Which tool? A quick decision tree

Am I typing code and want suggestions inline?
→ Completions — just keep typing, suggestions appear automatically
Do I have a question, need an explanation, or want a quick snippet?
→ Copilot Chat — ask it in the sidebar, fast and conversational
Is the task bigger than one file and I'm already in VS Code?
→ Copilot Agent Mode — Ctrl+I, give it a clear goal
Does the task require exploring my whole codebase, running shell commands, or working across many files?
→ Claude Code or Opencode — open the terminal
Is Claude unavailable or am I on a restricted network?
→ Opencode with GPT-4o or a local model

Real scenarios — which tool would you pick?

Scenario A
"I need to understand what this test does — there's no documentation and it's 80 lines long."
Answer
Copilot Chat — paste the test, ask "explain this test line by line". Quick, in-IDE, no file edits needed.
Scenario B
"I need to add API response mocking to our entire test suite — about 40 test files, all using different patterns."
Answer
Claude Code — needs to explore 40+ files, understand patterns, make consistent changes. This is exactly what terminal agents excel at.
Scenario C
"I keep getting a TypeScript error on this assertion and I don't know why."
Answer
Copilot Chat — paste the error, ask for help. Single file, quick fix. A terminal agent is serious overkill here.

Same task — two different tools

Task: "Create Playwright E2E tests for the checkout flow. Read existing auth tests as reference."
Copilot Agent Mode
1. Press Ctrl+I in VS Code to open Agent Mode
2. Type the task description into the chat window
3. Agent reads files visible in your workspace
4. Inline diffs appear in your editor — click Accept
5. Agent runs the tests, shows results in panel
Best when: you're already in VS Code and your codebase is well-structured
Claude Code
1. Open terminal, cd your-project && claude
2. Type the task — Claude reads the full repo structure first
3. Claude reads all related files, forms a plan
4. Approve each file edit or command as prompted
5. Claude runs tests, fixes failures, reports done
Best when: codebase is large or complex, or tasks span many files
Both tools can do the job. The difference is how much context they gather and where the workflow lives.

Using tools together

Example: building a new test suite for the payments module
1
Claude Code: "Scaffold the folder structure and base Page Objects for payments tests. Read the existing auth tests as reference."
2
Copilot Chat: "What's the Playwright way to intercept and mock a payment API response? Show me an example."
3
Copilot Completions: Writing individual test assertions inline, accepting suggestions as you type each one
4
Claude Code again: "Run the payment tests, identify any failures, and fix them. Don't change the test assertions, only fix the code."
These tools are complements, not alternatives. Use the right granularity for each part of the task.

When agents are overkill

Skip the agent — just do it yourself
· Single-line changes — renaming a variable, fixing a typo, updating a URL
· Understanding one function — Copilot Chat is faster and cheaper
· Copy-pasting a known pattern — if you already know what to write, just write it
· Writing a single test case — one test, clear scenario, just type it
· Quick config change — updating a port number, changing an environment variable
Common mistake
"I'll use Claude Code for everything — it's always better"
Reality
Starting a terminal agent session takes time. For a 30-second task, writing a 2-minute GCSV prompt and waiting for exploration is the wrong tradeoff. The agent is a power tool — use it for power tasks.

When agents are the obvious choice

️
Setting up a new test framework from scratch
Folder structure, config, base classes, fixtures, CI config — a half-day task done in one session
Migrating tests between frameworks
Cypress → Playwright, Jest → Vitest — 40 files, consistent conversion. A miserable afternoon vs. one session.
Adding a consistent pattern across many files
Adding error logging, updating selector conventions, wrapping all API calls — agent reads all files and applies consistently
Debugging a flaky test with complex async behaviour
Agent adds logging, runs multiple times, traces the failure pattern, proposes a fix — systematic where humans guess
️
Writing tests for an unfamiliar part of the codebase
Agent explores the code, understands it, then writes tests. You'd need to do the same exploration manually first.

Cost & access realities

GitHub Copilot
Flat monthly fee
Completions, Chat, Agent Mode all included. Predictable cost. Most organisations cover this as a standard developer tool.
Claude Code
Usage-based or subscription
Included with Claude.ai Max plan, or billed per token via Anthropic API. Long sessions on large codebases can add up — use for tasks that justify it by saving 30+ minutes.
Opencode
Free tool + model cost
The tool itself is free and open source. You pay for whichever model you configure. Can run local models via Ollama at zero marginal cost.

Building your daily AI workflow

Each task — the habit loop
1. Classify the task by scope → pick the right tool
2. For agents: commit first, write GCSV prompt
3. Review output: read the diff, check for red flags
4. Verify: run tests, confirm they pass meaningfully
5. Commit the result with a meaningful message
Weekly — maintain your toolkit
· Update CLAUDE.md with any new patterns or forbidden conventions you learned this week
· Note which tasks were good candidates for agents (and which weren't)
· Share useful prompts or patterns with the team — GCSV prompts are reusable
The team's collective CLAUDE.md and prompt library is a shared asset — keep contributing to it.
Start small: pick one task this week and use a terminal agent. Build the habit before trying to use it for everything.

Key takeaways

1
These tools are complements, not alternatives
Completions for inline typing, Chat for questions, Agent Mode or terminal agents for multi-file tasks. Use the right granularity for the job.
2
Know when NOT to use agents
Single-line changes, quick questions, and familiar patterns don't need an agent. Save agents for tasks that genuinely save 30+ minutes.
3
The decision is quick once you have the framework
Scope of task + where you're working + how complex = which tool. Internalize this and the choice becomes automatic.
4
Build the habit deliberately
Pick one task this week. Use the GCSV formula. Commit first, review the output. The habit compounds — each session makes the next one easier.
