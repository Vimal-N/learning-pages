What is GitHub Copilot?

GitHub Copilot is an AI coding assistant that lives inside your editor. It watches what you're writing and suggests what comes next — in real time. It also has a chat interface where you can ask questions about your code, get explanations, and request changes.

BUILT BY
GitHub (Microsoft) & powered by OpenAI. Deeply integrated into VS Code, JetBrains IDEs, and GitHub.com.
TRAINED ON
Billions of lines of public code from GitHub repositories. This is why it understands frameworks, patterns, and idioms so well.

GitHub Copilot has two modes

Mode 1: Inline Completions
Passive. Copilot watches what you type and suggests what comes next — inline, greyed out. Tab to accept.
test('user can log in', async ({ page }) => {
  await page.goto('/login');
  // ← You type here, Copilot suggests the rest
  await page.fill('#email', 'test@example.com');
  await page.fill('#password', 'password123');
  await page.click('#submit');
Best for: fast boilerplate, repetitive patterns, completing what you started
Mode 2: Copilot Chat
Active. Open a chat panel and have a conversation about your code. Ask questions, request changes, get explanations.

Setup Step 2 of 3

Sign in & activate

1
Click the Copilot icon in the status bar
VS Code will prompt you to sign in to GitHub. Click 'Sign in to GitHub'.
2
Authorise in the browser
A browser tab opens. Click 'Authorize GitHub Copilot'. Return to VS Code — it will detect the authorisation automatically.
3
Confirm Copilot is active
Open any code file and type a comment like // test that user can log in. If Copilot is working, a suggestion will appear.
Common issue: Copilot icon shows an exclamation mark — usually means your GitHub account doesn't have a licence assigned. Check with your GitHub admin or the GitHub Copilot settings at github.com.

Your first inline completion

Open a .ts file and type this — then pause:
// Test: user submits the contact form successfully
test(
What Copilot suggests (greyed out — hit Tab to accept):
'user submits the contact form successfully', async ({ page }) => {
  await page.goto('/contact');
  await page.fill('[name="name"]', 'John Smith');
  await page.fill('[name="email"]', 'john@example.com');
  await page.click('button[type="submit"]');
  await expect(page.getByText('Message sent!')).toBeVisible();
});
→ Copilot reads your comment, infers the framework from surrounding code, and generates the whole test. Always review before keeping — check selectors are correct for your actual app.

How inline completions work

👁️
It reads your surrounding context
Copilot looks at the code above and around your cursor — your imports, function names, comments, and patterns already established in the file.
💭
It predicts what comes next
Using patterns from its training data, it generates the most likely continuation of what you're writing. Shown in grey, dimmed text.
⌨️
You accept or dismiss
Tab — accept the full suggestion. Esc — dismiss. Alt+] / Option+] — see alternate suggestions. Ctrl+→ — accept word by word.
Better context = better suggestions. Meaningful variable names, descriptive comments, and consistent patterns all improve what Copilot offers.

What Copilot can do

FOR QA ENGINEERS
  • ✓ Write test boilerplate from a description
  • ✓ Generate test data (valid and invalid)
  • ✓ Explain what a piece of code does
  • ✓ Suggest fixes for failing tests
  • ✓ Refactor repetitive test patterns
  • ✓ Add or improve code comments
GENERAL CAPABILITIES
  • ✓ Complete any code pattern you start
  • ✓ Answer questions about libraries and APIs
  • ✓ Translate between code patterns
  • ✓ Write utility functions and helpers
  • ✓ Suggest regular expressions
  • ✓ Help with Git commit messages and PR descriptions

What Copilot cannot do

✗
Know your actual application
It doesn't have access to your live app. Selectors, URLs, and data it generates are educated guesses — verify them.
✗
Run or execute code
Copilot reasons — it doesn't run. It can suggest a fix but can't verify the fix works. You have to run the test.
✗
Know your team's conventions
Unless you tell it (or it infers from surrounding code), it doesn't know your naming conventions, folder structure, or architectural decisions.
✗
Be right about recent library versions
APIs change. Copilot's training data has a cutoff. For any Playwright or framework API it suggests, verify against the official docs.

Privacy & code sharing

Individual Plan
Code snippets may be sent to GitHub/OpenAI for processing. You can opt out of telemetry in VS Code settings → GitHub Copilot → Telemetry.
Business Plan
Code is NOT used for training by default. Prompts and suggestions are not retained by GitHub. Enterprise adds extra controls.
Always follow these rules regardless of plan:
  • Never paste API keys, passwords, or credentials
  • Never paste production customer data or PII
  • Check your org's AI usage policy before using on sensitive codebases

Key Takeaways

What to take from today

1
Copilot has two modes: Inline Completions (passive, watches and suggests) and Chat (active, you ask questions). Learn to use both.
2
Better context in the file = better suggestions. Meaningful names, comments, and established patterns all help Copilot.
3
Copilot doesn't know your actual app. Selectors, URLs, and API calls it generates are educated guesses — always verify and test.
4
On Business/Enterprise plans, your code is not used for training. On Individual plan, opt out of telemetry in settings if needed.
5
Never paste credentials, PII, or secrets into Copilot. Check your org's AI policy before using on sensitive codebases.
