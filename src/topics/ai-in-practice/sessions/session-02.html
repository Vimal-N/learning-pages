Era 1  ·  1960s – 2010s

Rule-Based Chatbots

If you typed the right thing, they worked. If you didn't — good luck.

HOW THEY WORKED
  • → Match input against a list of patterns
  • → Return the pre-written response for that pattern
  • → No real understanding of language
  • → Break instantly outside their decision tree
EXAMPLES
  • → ELIZA (1966) — first chatbot, pattern matching
  • → Bank IVR bots — "Press 1 for balance"
  • → Website FAQ bots of the 2010s
  • → Help desk triage bots
The core problem: any input outside the script = system failure. Engineers spent months writing decision trees that users broke in 5 minutes.
Era 2  ·  2011 – 2022

Voice Assistants & NLP

A genuine step forward — but still a very long way from understanding you.

2011
Siri
Apple — intent matching on device
2014
Alexa
Amazon — skills ecosystem

Google Assistant
Google — knowledge graph backed
WHAT IMPROVED VS WHAT DIDN'T
Better
No longer need exact keywords — natural phrasing works
Still broken
Anything outside a pre-known intent → "I can't help with that"

Why the old approach hit a wall

Humans are unpredictable
You can't pre-write a response for every possible question. Language is infinite. Rule lists are finite.
Maintenance cost grew exponentially
Every new capability required engineers to write more rules. Bots became unmaintainable nightmares.
Context blindness
These systems had no memory. Every message was independent. They couldn't hold a real conversation.
Domain locked
A flight chatbot knew about flights. Ask it about the weather and it broke. Knowledge couldn't transfer.
Era 3  ·  2020 – present

The Generative Leap

Jun 2020
GPT-3
First large language model with truly impressive capabilities — but API only, no public UI
Jun 2021
GitHub Copilot Preview
First AI coding assistant — developers could get code suggestions inside VS Code
Nov 2022
ChatGPT launches
1 million users in 5 days. Generative AI became a mainstream conversation overnight
2023–2025
Explosion of tools
Claude, Gemini, Copilot Chat, Cursor, Codex, agents — the ecosystem expanded rapidly

What actually changed

Old approach: Scripted
  • Response is pre-written by a human
  • Match input to a known pattern
  • Anything off-script = failure
  • No memory between turns
  • Domain-locked
New approach: Generative
  • Response is generated fresh each time
  • Understands intent, not just keywords
  • Handles novel questions gracefully
  • Maintains context across the conversation
  • Generalises across domains
2021 onwards

Coding Assistants — AI enters the editor

WHAT CHANGED
  • → AI moved inside your development environment
  • → Real-time inline code suggestions as you type
  • → Trained on millions of real code repositories
  • → Understands context from surrounding code
TOOLS IN THIS ERA
  • → GitHub Copilot (2021) — the original
  • → Copilot Chat (2023) — conversational layer
  • → Cursor — AI-native editor
  • → Codeium, Tabnine — alternatives
For QA engineers: AI could now write test stubs, generate assertions, suggest test data, and explain unfamiliar code — without leaving VS Code.
Era 4  ·  2024 – present

Agents — AI that takes action

Chatbot / Assistant
You ask → it answers. One exchange at a time. No persistent action.
️
Coding Assistant (Copilot)
You type → it suggests. Real-time help inside your editor. Still one suggestion at a time.
Agent (Claude Code, Copilot Agent Mode)
You describe a goal → it plans, uses tools, reads files, writes code, runs tests, iterates. Autonomous multi-step execution.

The agentic loop

STEP 1
Receive goal
User describes what they want done
️
STEP 2
Plan
LLM decides which steps to take
️
STEP 3
Act & observe
Runs a tool, reads output, adjusts
STEP 4
Repeat or finish
Loops until goal is achieved
Example: "Write Playwright tests for the login page" → agent reads the login page HTML, generates test file, runs the tests, sees 2 failures, reads the errors, fixes the selectors, runs again, all pass. Done.

What hasn't changed

STILL YOUR JOB
  • Deciding what to test and why
  • Identifying edge cases and risk areas
  • Verifying AI-generated code is correct
  • Understanding the system under test
  • Interpreting results with business context
AI HANDLES MORE OF
  • → Writing the boilerplate test code
  • → First-pass debugging of failures
  • → Explaining unfamiliar code
  • → Generating repetitive variations
  • → Documentation and comments
Myth

Agents are fully autonomous — set them loose and come back when it's done

Reality
Agents work best with clear, scoped tasks and regular human checkpoints. They can go off track, make incorrect assumptions, and produce output that looks right but isn't. Think of an agent as a fast junior engineer — capable of doing a lot, but still needs your direction and review.
Good use: "Write Playwright tests for the login page — here's the page URL and the acceptance criteria"
Risky use: "Fix all the bugs in our test suite" — too vague, too much scope, no checkpoints
Key Takeaways

What to take from today

1
Old chatbots worked by matching inputs to scripts. Modern LLMs generate responses — that's a fundamentally different capability.
2
The accessibility barrier broke in November 2022. The technology had been building for years — ChatGPT made it available to everyone.
3
We're now in the agent era — AI that doesn't just answer but takes action, uses tools, and completes multi-step tasks autonomously.
4
Your domain expertise as a QA engineer is more valuable, not less. AI executes faster — you still set direction and verify results.
5
Agents need scoped tasks, clear context, and human checkpoints. They are fast junior engineers, not infallible autonomous systems.
