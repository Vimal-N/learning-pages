The Evolution — From Chatbot to Agent

The five eras of AI tools, what the Transformer breakthrough changed, and what "agent" actually means in practice.

Session 02  ·  Phase 1: AI Foundations  ·  Beginner

What you'll get from this session
By the end of Session 02 you will be able to trace how AI tools evolved from rule-based chatbots to autonomous agents, explain what changed with the Transformer architecture, and use the word "agent" correctly when you hear it in demos or job descriptions.

The Five Eras of AI Tools

A timeline from rule-based bots to autonomous agents — with the key shift at each stage.

ERA 1
Rule-Based Bots
~2010–2017
If-then decision trees. FAQ bots. "Press 1 for billing." Could only respond to exact match phrases. Not AI in the modern sense — just branching logic dressed up as conversation.
ERA 2
Early NLP Models
~2017–2020
BERT, early GPT. The first NLP (Natural Language Processing) models — AI that could understand intent and context in a sentence. Power-user territory — APIs and research. Not yet consumer-friendly. Useful but limited to narrow, single-purpose tasks.
ERA 3
General LLMs
~2020–2022
GPT-3, Codex. Could hold open-ended conversations, write code, summarise text. Still text-in, text-out. The first era where non-developers could get real value without engineering expertise.
ERA 4
Coding Assistants
~2022–2023
GitHub Copilot, ChatGPT. LLMs (Large Language Models) embedded directly in your editor. Real-time suggestions as you type. The first time AI felt like a genuine co-worker rather than a search engine.
ERA 5
Agents
~2024–now
Claude Code, Copilot Agent Mode, Cursor. LLMs that can take actions — read files, run code, search the web, call APIs — in a loop until a task is complete. This is where the real productivity leap is.

Each Era — Up Close

What each era actually looked like day-to-day — with real examples a QA engineer would recognise.

ERA 1 Rule-Based Bots ~2010–2017
A flow chart disguised as a chatbot. Every possible question was pre-written, and every answer was pre-decided by a human.
Example
You visit a bank's website. A chat bubble appears: "Hi! Type 1 for account balance, 2 for lost card, 3 to speak to an agent."
You type: "I need to dispute a transaction."
Bot replies: "I didn't understand that. Type 1 for account balance, 2 for lost card…"
It couldn't handle anything outside its script. No understanding — just pattern matching on keywords.
ERA 2 Early NLP Models ~2017–2020
NLP (Natural Language Processing) models that understood intent and context — but trained on narrow domains and not yet accessible to everyday users.
Example
Gmail's Smart Reply ("Sounds good!" / "Thanks!" suggestions under emails). Google's search autocomplete. YouTube's automatic captions.
Good at one narrow task each. You could not have a conversation with them or ask them to write a test.
ERA 3 General LLMs ~2020–2022
You could now have an open-ended conversation and get genuinely useful text back — including code.
Example
You open ChatGPT (GPT-3.5) and type: "Explain what a CSS selector is in plain English." It explains it clearly. You type: "Now write me a simple JavaScript function that counts how many times a button has been clicked." It writes one.
First time non-developers got real value. But you had to write code yourself — it only produced text output.
ERA 4 Coding Assistants ~2022–2023
The LLM (Large Language Model) moved inside your editor. It watched you type and suggested what came next — like a very fast, very knowledgeable pair programmer.
Example
You open VS Code with GitHub Copilot installed. You type the comment: // Test: login with wrong password should show error then press Enter. Grey "ghost text" appears — a full Playwright test — before you've typed a single line of code. You press Tab. Done.
For the first time, the AI was in your workflow, not a separate tab.
ERA 5 Agents ~2024–now
The AI stopped just generating text and started taking actions — reading files, running code, fixing errors — then doing it all again until the job was done.
Example
You open Copilot Agent Mode and type: "Add Playwright tests for the checkout flow. Match the style of the existing tests."
Without further input, it reads your test files, reads your config, writes a new test file, runs it in your terminal, reads the error output, fixes the selector that was wrong, runs it again, and tells you it passed.
You reviewed what it did — you didn't do it yourself.

The Transformer Breakthrough

One architecture change in 2017 unlocked everything that followed. Here's what actually changed.

Before 2017, AI language models processed text sequentially — one word at a time, with limited memory of what came before. The Transformer architecture (introduced in a 2017 Google paper titled "Attention Is All You Need") changed this fundamentally.

Before Transformers (RNNs)
  • Read text word by word, left to right
  • Earlier context "faded" as sequences got longer
  • Slow to train — couldn't parallelise
  • Lost quality on long documents
With Transformers
  • Processes all tokens simultaneously
  • Attention mechanism: learns which parts of the input matter for each output
  • Scales with more data and more compute
  • Can handle very long contexts without degrading

The key insight: with attention, the model can learn "when I see the word test, the word assertion a few tokens later is probably important" — across the entire input simultaneously. Scaled to billions of parameters and trained on the internet, this unlocked general-purpose language understanding.

You don't need to understand the maths. What matters is this: the Transformer breakthrough is why GPT-3, GPT-4, Claude, and Gemini exist. The 2017 paper made scaling possible, and scaling made quality possible.

What "Agent" Actually Means

The word gets overused. Here's the precise definition — and why it matters for how you work with these tools.

The word "agent" is used loosely in marketing material. Here is the precise technical meaning: an AI agent is an LLM that has been given access to tools — functions it can call to take actions in the world — and runs in a loop until a task is complete.

1
LLM core
The same token prediction we discussed in Session 01. This is the "brain" — it understands your task and decides what to do next.
2
Tools
Functions the model can call: read a file, write a file, run a terminal command, search the web, call an API. The LLM decides when to use them.
3
The loop
Think → act → observe result → think again → act again. It keeps going until the task is done or it gets stuck. Unlike Chat, which is one prompt, one response.
NOT an agent

"Write me a Playwright test for the login page." — This is a single-turn chat prompt. It generates text. You copy it. Nothing was executed.

IS an agent

"Add a Playwright test for login failure." — An agent reads your existing tests for style, reads the page source for selectors, writes the test file, runs it, reads the error, fixes it, runs it again.

Agents in Practice

What an agent actually does during a real task — step by step.

Here's what happens when you give Copilot Agent Mode a task like "add a test for the checkout flow":

→
Read existing tests — scans your test files to understand naming conventions, what assertions you use, how Page Objects are structured
→
Read config files — looks at playwright.config.ts to find your base URL, timeout settings, project structure
→
Write the test — generates a test file that matches your project's style, using the patterns it found
→
Run the test — executes it in your terminal, reads the output
→
Fix the error — if the test fails, reads the error and iterates until it passes (or until it needs to ask you for something it can't determine)

Critical point: An agent can modify files, run commands, and make decisions — all without asking you each time. This is powerful and requires a different kind of oversight than Chat. You review what it did, not just what it said. Session 06 and Session 13 cover this in detail.

The Spectrum of AI Tools Today

From passive suggestion to autonomous action — where each tool sits.

You'll hear many tool names across this course. They sit on a spectrum from passive (makes suggestions) to active (takes actions):

Passive suggestion Autonomous action
Inline Completions
Passive
Copilot ghost text. Suggests next tokens as you type. You accept or dismiss. Zero autonomy.
Chat
Conversational
Ask a question, get a text response. You copy code manually. One turn at a time. (Copilot Chat, Claude, ChatGPT)
Copilot Agent Mode
Semi-autonomous
Reads files, writes edits across multiple files, runs terminal commands. Shows you proposed changes before applying.
Claude Code / Opencode
Fully autonomous
Terminal-based agents. Full access to your filesystem and shell. Reads, writes, runs, and iterates autonomously. Highest capability — and highest need for oversight.

This course works through the spectrum: Phases 1–2 cover the passive end (completions, chat), Phase 3 covers agent mode, Phases 4–5 go into terminal agents and multi-agent workflows. You'll start with the safest tools and build up.

What This Means for QA Work

How the evolution changes your role — and what stays the same.

What changes
  • Writing tests gets faster — agents can draft an entire test suite from a description
  • Debugging is faster — agents can read error output and iterate on fixes
  • Repetitive work (data factories, helper utilities) can be delegated
  • A single QA engineer can cover more ground with less time on boilerplate
What doesn't change
  • You still decide what to test and what risk level is acceptable
  • You still verify that generated tests actually work and actually prove something
  • You still understand the product — AI doesn't know your app
  • You still own the quality judgement call

The mindset shift: Moving from chat to agent requires trusting the tool more — but verifying it differently. Instead of checking one code block, you review what it changed across your project. The oversight changes shape, it doesn't disappear. This is why Session 06 (Agent Mode) spends time on reviewing changes safely.
