The Full Spectrum

Five tools, five different trade-offs. Each one optimised for a different task type.

Inline Completions
Ghost text as you type
Best for: code you're already writing — completing a function you started, suggesting the next assertion, filling in boilerplate. Not for: anything requiring project understanding or multi-step tasks. Speed: instant. Oversight needed: minimal (you see it before accepting).
Chat (Copilot Chat / Claude / ChatGPT)
Conversational, one turn
Best for: questions, explanations, drafting a prompt, understanding unfamiliar code, getting a first version of something to review. Not for: tasks requiring file writes or running code. Speed: seconds. Oversight needed: read before using.
Copilot Agent Mode
Editor-scoped autonomy
Best for: tasks within your VS Code workspace — adding a feature, editing multiple related files, small refactors. Shows proposed changes before applying. Not for: large multi-directory tasks. Speed: minutes. Oversight needed: review diffs per step.
Claude Code / Opencode
Terminal — full autonomy
Best for: larger tasks — building a full test suite, major refactors, investigating a complex failure across many files, generating documentation. Not for: tiny tasks where overhead isn't worth it. Speed: minutes to tens of minutes. Oversight needed: git diff review at the end.

Decision Framework

Use this grid to decide which tool to reach for on any QA task.

Inline completions (Copilot ghost text)
I need to understand something — what a function does, why a test is failing
Chat (Copilot Chat, Claude, ChatGPT)
I want a first draft of something to review and edit
Chat — paste result and refine it yourself
I want AI to write and apply a change in my editor, within one project
Copilot Agent Mode
I need to add tests for one feature — about 1 new spec file
Copilot Agent Mode or Claude Code — either works
I need to build or refactor tests across many files or features
Claude Code or Opencode (terminal agent)
I need to run commands, read error output, and iterate automatically
Terminal agent — agents can close the loop themselves
I'm on a machine or CI where I can't use VS Code
Terminal agent — headless, runs anywhere Node is available

QA Scenarios — Which Tool?

Real situations from QA work — and which tool fits each one best.

SCENARIO A test is failing with a confusing error
You need to understand the error before deciding how to fix it.
Best tool
Chat. Paste the error + test code into Copilot Chat or Claude. Ask for a diagnosis. Read the answer, verify it makes sense, then apply the fix yourself (or ask Agent Mode to apply it). Don't open a terminal agent just to diagnose — it's slower and the extra overhead isn't needed for a single file fix.
SCENARIO A new feature has 20 acceptance criteria and needs full test coverage
Large scope — multiple test files, multiple scenarios, needs to match existing project style.
Best tool
Terminal agent (Claude Code / Opencode) with GCSV task prompts. Break the AC into 3–4 task chunks (one feature area at a time), run each task separately, review the diff and run tests between each. This is exactly the use case terminal agents are designed for — large multi-file tasks that would take an hour by hand.
SCENARIO You're writing a Page Object and want suggestions for each method
You're actively typing — you want AI to complete what you start, not take over.
Best tool
Inline completions. Start typing the method signature — async fillLoginForm( — and let Copilot ghost text suggest the implementation. Tab to accept, tweak if needed. No prompt, no chat window — just Tab. Fastest for code you're already writing.
SCENARIO You want to refactor 3 test files to use a new shared fixture
A focused change across a small number of files — bounded scope.
Best tool
Copilot Agent Mode. Open the 3 files in VS Code. Use Agent Mode with a clear scope statement. It shows you the proposed changes before applying. For 3 files, the step-by-step approval model of Agent Mode is appropriate — you don't need the full terminal agent overhead.

Cost and Capability Trade-offs

More capability costs more — in tokens, in time, and in oversight effort.

Lower cost — lower capability
  • Inline completions: included in Copilot subscription, instant, zero setup
  • Chat: low token cost per turn, no file access, you control what's shared
  • Best for: small, contained tasks — understanding, drafting, single-file edits
Higher cost — higher capability
  • Agent Mode: more tokens per session (reads multiple files), Copilot subscription or per-token
  • Terminal agents: highest token usage — reads entire project context, long loops
  • Best for: large tasks that justify the investment

The cost-to-value question

Before opening a terminal agent, ask: would this task take me more than 30 minutes by hand? If yes, the agent session is worth the token cost. If it would take 5 minutes, open a chat window instead — faster, cheaper, and less to review.

Claude Max and Copilot Enterprise users: If you're on a flat-rate subscription (Claude Max or GitHub Copilot Enterprise), the per-token cost calculus changes — you've already paid. In that case, the main cost is your review time. Still decompose tasks into reviewable chunks — not because of billing, but because smaller tasks are easier to verify and recover from.
