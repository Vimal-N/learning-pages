# Chat Variables

Context attachments — controls exactly what content Copilot can see in your message
Mental Model

Without #, Copilot sees your open file and guesses the rest. With #variable, you hand it exactly the right material — a specific file, your highlighted code, or the terminal output — so its answer is grounded in your actual context, not a generic approximation. Think of it as the difference between telling a colleague "my test is broken" versus handing them the error, the test file, and the failing assertion together.

Code & File References
#file
Attaches the full content of a specific file. Type #file and a picker lets you choose from your workspace. The file doesn't need to be open.
Review the patterns in #file and suggest the missing edge cases
#selection
Attaches whatever you have highlighted in the editor. Most precise variable — scopes to exactly the lines you selected, nothing more.
Why does #selection throw a timeout in CI but pass locally?
#editor
Attaches the entire active editor tab. Useful when you want Copilot to consider the full file, not just a selection, without using @workspace scope.
Look at #editor — what error-state assertions are we missing?
#codebase
Broad search across your whole codebase. Similar to @workspace but used inline in a message rather than switching participant mode entirely.
Does #codebase already have a helper for generating test fixtures?
Terminal References
#terminalLastCommand
Attaches the output of the last command you ran in the integrated terminal. The fastest way to get a diagnosis from a test failure — no copy-pasting a stack trace.
@terminal #terminalLastCommand — explain this failure and what's causing it
#terminal
Attaches the full terminal buffer — everything currently visible in the terminal panel, not just the last command. Useful when the failure spans multiple commands.
Based on #terminal, which tests are failing and what do they share?
Source Control & Other
#git
Attaches recent git history — useful for understanding what changed and generating meaningful commit messages from actual diffs.
Based on #git, write a clear commit message for these changes
#fetch evolving
Fetches content from a URL and attaches it inline. Reference live documentation or API specs without leaving the chat.
#fetch https://playwright.dev/docs/api how do I fix this locator?
#problems evolving
Attaches problems from the VS Code Problems panel — lint errors, type errors, and diagnostics all at once.
Write tests that would catch #problems
💡 GitHub confirmed as of May 2025 that some # variables like #fetch, #new, and #problems exist but are not fully documented yet. Always type # in VS Code to see the full current list available in your installed version — the autocomplete picker is the authoritative source.
