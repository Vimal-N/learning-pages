Power Combinations

Real QA scenarios using @, #, and / together — the patterns that save the most time
@participant Who Copilot acts as #variable What Copilot can see /command What task to perform
1
Diagnose a failing test in CI without copy-pasting

Prompt

@terminal #terminalLastCommand explain why this test failed and what the stack trace is pointing to

Why it works

→ @terminal gives Copilot terminal context. #terminalLastCommand attaches the actual test runner output. Together they give a targeted diagnosis grounded in your real error — not a generic "check your selectors" response.
2
Find test coverage gaps across the whole project

Prompt

@workspace which Playwright tests are missing assertions on the error state or loading state?

Why it works

→ @workspace searches every test file in the indexed project — not just the open tab. You get an actual cross-file gap analysis rather than a per-file answer.
3
Generate tests that match your project's actual patterns

Prompt

/tests #file checkoutFlow.ts — focus on async state changes and network error handling

Why it works

→ /tests sets the task unambiguously. #file grounds the output in your actual code — selectors, import style, assertion patterns — so Copilot generates tests that fit your suite rather than generic boilerplate.
4
Understand a complex selector you inherited

Prompt

/explain #selection — tell me why this locator might fail after a DOM update

Why it works

→ /explain requests an explanation. #selection pins it to exactly the highlighted lines — not the whole file. Copilot explains the fragility of the specific selector strategy you're looking at.
5
Find how mocks are set up without reading every file

Prompt

@workspace where do we set up API mocks before tests? Show me the pattern we use

Why it works

→ Returns the actual patterns in your codebase — real file paths, real method calls. When you then ask Copilot to generate new test setup, it replicates your team's existing convention rather than inventing one.
