@ Chat Participants

Domain specialists — controls who Copilot acts as when you ask a question
Mental Model

When you type @participant, you are switching Copilot into a specialised mode. Without an @, Copilot only looks at your open file. With one, it gains access to a broader domain — your whole workspace, VS Code's settings system, or your terminal session.

@workspace
Indexes and searches across all files in your open project. Best participant for project-wide questions — finding patterns, locating utilities, understanding test coverage across the whole codebase.
@workspace where do we mock API calls in this project?
@vscode
Knows about the VS Code editor itself — settings, keybindings, commands, launch configs, and extension features. Use this when your question is about the tool, not the code inside it.
@vscode how do I configure the Playwright test runner in launch.json?
@terminal
Knows about the integrated terminal — its shell, command history, and buffer output. Pairs perfectly with #terminalLastCommand to diagnose failures without copy-pasting.
@terminal explain the error from the last command I ran
@github github.com only
Accesses GitHub repositories, issues, and pull requests on github.com. Not available inside VS Code — only works in the Copilot Chat panel on the GitHub website.
@github find open issues labeled "flaky test" in this repo

When to use each participant

Participant Reach for it when you want to… QA example
@workspace Find something across the whole project — page objects, mock setup, existing helpers "Do we already have a helper that generates test users?"
@vscode Configure the editor, understand settings, or learn a VS Code shortcut "What setting controls test retries in the test explorer?"
@terminal Diagnose a terminal error or understand a shell command without leaving VS Code "Why did my npm test script fail? What does this exit code mean?"
@github Search issues and PRs from the GitHub website chat panel "Which open issues in this sprint are about broken selectors?"
💡 Pro tip: You can combine a participant with a # variable in the same message. For example: @workspace which tests cover the changes in #file — Copilot searches the whole project, but focuses its answer around the file you attached.
