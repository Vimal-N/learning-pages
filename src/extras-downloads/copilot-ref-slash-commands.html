/ Slash Commands

Task shortcuts — precise instructions without writing out the full request every time
Mental Model

Slash commands are unambiguous shortcuts. Instead of writing "can you look at this function and generate unit tests for it covering edge cases", you write /tests and Copilot knows exactly what you want. They remove ambiguity and consistently produce the right output type. Combine them with # variables to scope exactly what gets generated.

/explain
Explains how selected code works in plain English. Invaluable for inherited Playwright selectors, complex regex patterns, or unfamiliar utility functions you didn't write.
QA use /explain #selection — why might this selector break after a DOM update?
/fix
Diagnoses an error and proposes a fix. Works on selected code or the full active file. Best practice: let it explain the diagnosis first, then evaluate whether the fix makes sense before applying.
QA use /fix — this assertion fails because the modal loads asynchronously
/tests
Generates test cases for the selected function or active file. Add context with #file or a description of what edge cases to cover — the more specific, the better the output.
QA use /tests #file — cover async state changes and network error handling
/doc
Generates inline documentation — JSDoc comments, Python docstrings, or TypeScript descriptions — for the selected function or class. Useful before a PR review.
QA use /doc this test helper so teammates know what parameters it expects
/new
Scaffolds a new project or file structure. Describe what you want to create and Copilot generates the folder layout, config files, and boilerplate.
QA use /new Playwright project with Page Object Model structure and global auth setup

Combine commands with # variables

Generate tests scoped to one file
/tests #file checkoutFlow.ts
— include null inputs and network timeout cases
Fix a failure shown in terminal
/fix #terminalLastCommand
— suggest what changed and why this broke
Explain just the highlighted code
/explain #selection
— focus on why this assertion might always pass
Document a helper across a file
/doc #editor
— add JSDoc to every exported function
